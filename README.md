OrderSystem

Система обработки заказов с асинхронным взаимодействием сервисов через RabbitMQ и отправкой email-уведомлений о статусе заказа.
Пользователь может создать заказ через Swagger UI. После этого сервисы обмениваются сообщениями через RabbitMQ и отправляют пользователю письмо со статусом заказа.

Функциональность

— создание заказа через API
— асинхронная обработка заказов
— обмен сообщениями между сервисами через RabbitMQ
— отправка email-уведомления о статусе заказа
— тестирование API через Swagger

При создании заказа можно указать один из товаров:

— Puzzle
— Socks
— Notebook
— Cup
— Pot

Также необходимо указать:

— количество товара
— email пользователя

После обработки заказа пользователь получает письмо со статусом заказа.

Архитектура

— Сервисы взаимодействуют асинхронно через брокер сообщений RabbitMQ.
— Процесс обработки заказа:
  — Пользователь создаёт заказ через API.
  — Заказ отправляется в очередь RabbitMQ.
  — Другой сервис обрабатывает сообщение и посылает в ответ свое.
  — Пользователь получает email со статусом заказа.

Технологии

— .NET / ASP.NET Core
— RabbitMQ
— Docker
— Swagger (OpenAPI)

Запуск проекта

— Необходимо установить Docker
— Клонировать репозиторий:
    git clone https://github.com/ekaterinaryzhenkova/OrderSystem.git
    cd OrderSystem
— Запустить все сервисы: docker-compose up --build

После запуска API будет доступно по адресу: http://localhost:5001/swagger/index.html
